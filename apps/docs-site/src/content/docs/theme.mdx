## Colors

Each token has light and dark values keyed by the same semantic name. Components read tokens, not raw hex values, so dark mode is automatic.

```ts title="src/theme/colors.ts"
export type ColorTokens = {
  background: string;
  surface: string;
  surfaceMuted: string;
  border: string;
  borderStrong: string;
  foreground: string;
  foregroundMuted: string;
  foregroundSubtle: string;
  primary: string;
  primaryForeground: string;
  primaryMuted: string;
  destructive: string;
  destructiveForeground: string;
  success: string;
  warning: string;
  ring: string;
  overlay: string;
};
```

## Typography

`fontSize`, `lineHeight`, and `fontWeight` keyed by xs–4xl.

## Shadows

`Platform.select` picks iOS `shadow*` values, Android `elevation`, and falls back to iOS for web. Three steps: `sm`, `md`, `lg`.

## Consuming tokens

Declare `createStyles` at module scope. It builds the StyleSheet once per scheme (light/dark) and caches it — the closure runs at most twice per component lifetime, not on every render.

```tsx
import { createStyles } from "@/utils/use-styles";

const useStyles = createStyles((t) => ({
  base: {
    backgroundColor: t.colors.surface,
    borderColor: t.colors.border,
    borderRadius: t.radius.lg,
    padding: t.spacing.lg,
  },
}));

function Card() {
  const styles = useStyles();
  return <View style={styles.base} />;
}
```

For prop-derived values (e.g. an avatar's size-dependent `borderRadius`), put the static styles in `createStyles` and inline the dynamic parts:

```tsx
const useStyles = createStyles((t) => ({
  base: { backgroundColor: t.colors.primaryMuted, alignItems: "center" },
}));

function Avatar({ size }) {
  const styles = useStyles();
  return <View style={[styles.base, { width: size, height: size, borderRadius: size / 2 }]} />;
}
```

A small `cn(...)` helper is included in `utils/cn.ts` for assembling style arrays without the booleans-in-arrays noise:

```tsx
import { cn } from "@/utils/cn";

<View style={cn(styles.base, pressed && styles.pressed, disabled && styles.disabled, style)} />
```
